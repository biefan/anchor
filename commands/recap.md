---
description: 复盘当前任务进度——已做 / 剩余 / 岔路 / 需要决策的点
argument-hint: ""
---

按 `ec` skill 的"复盘节点"原则汇报当前状态。**只汇报，不推进**——给用户/自己一个清晰快照。

### 1. 跑 TaskList 拿当前所有 task

按 status 分桶汇总：

```
✅ 已完成（N 项）
  - #1 [completed]: ...
  - #2 [completed]: ...

🔄 进行中（M 项）
  - #3 [in_progress]: ...

⏳ 待办（K 项）
  - #4 [pending]: ...
  - #5 [pending]: ...
```

### 2. 总结本会话主要改动

简短列出已修改 / 新建的关键文件（如有，最多 5 个，按重要性）。

### 3. 列出岔路 / 需要决策的点

- 走到哪个 task 时碰到了岔路？两个/多个方案是什么？
- 还有哪些用户没明确表态、当前按默认假设走的决策？

### 4. 列出范围外发现

会话过程中发现但**没改**的范围外事项（这是好习惯——按 SKILL.md "偏题刹车"原则记下来）：
- "顺手发现 X 有 bug" 等
- "可优化点 Y"

让用户决定要不要加成新 task。

### 5. 推荐下一步

基于上面快照，建议一句话：
- "下一步：跑 `/next` 推进 task #4"
- 或 "建议先 `/done` 因为剩余 task 实际都是 nice-to-have"
- 或 "建议先决策 X，再继续"

**重要**：本命令不修改任何状态。只汇报。用户/Claude 看完决定下一步。
